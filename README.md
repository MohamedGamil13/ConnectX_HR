# Connect X HR Attendance System

> A Flutter app for managing student attendance via QR codes, AI-powered session management, and Google Sheets integration — built entirely on free-tier services. No credit card required.

---

## What this app does

The Connect X HR system automates summer training attendance tracking from end to end:

1. Admin creates a session (manually or by asking the AI in plain language)
2. The app generates a unique, signed QR code for every student and emails it to them
3. Admin scans QR codes at the session to mark attendance instantly
4. All attendance is auto-synced to Google Sheets and visualized in-app

## Setup

### 1. Clone and install

```bash
git clone https://github.com/your-org/connectx-hr.git
cd connectx-hr
flutter pub get
```

### 2. Firebase

```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Connect to your Firebase project
flutterfire configure
```

In Firebase Console:
- Enable **Authentication → Email/Password**
- Enable **Cloud Firestore** (production mode)
- No need to upgrade beyond Spark plan

### 3. Google Sheets API

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Enable **Google Sheets API**
3. Create a **Service Account** → download the JSON credentials
4. Place it at `assets/sheets_credentials.json`
5. Share your Google Sheet with the service account email

### 4. Environment keys

On first admin login, the app prompts for:
- Resend API key (from resend.com)
- Gemini API key (from aistudio.google.com)
- JWT secret (generate any secure random string)
- Google Sheet ID (from the sheet URL)

All stored securely via `flutter_secure_storage` — never hardcoded.

## QR security design

Each QR encodes a JWT with this payload:

```json
{ "sub": "student_001", "sid": "session_003", "exp": 1720000000, "jti": "uuid-v4" }
```

**On scan, the app validates:**
1. Signature — was this signed with our secret?
2. `exp` — has the QR expired?
3. `sid` — does this match the currently active session?
4. `jti` — has this exact token already been scanned? (prevents reuse)

Only if all four checks pass is attendance marked.

## Firestore security rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    match /sessions/{id} {
      allow read: if request.auth != null;
      allow write: if request.auth.token.role == 'admin';
    }

    match /students/{id} {
      allow read: if request.auth != null;
      allow write: if request.auth.token.role == 'admin';
    }

    match /attendance/{id} {
      allow read: if request.auth != null;
      allow write: if request.auth.token.role == 'admin';
    }

    match /qr_tokens/{id} {
      allow read, write: if request.auth.token.role == 'admin';
    }
  }
}
```

## AI assistant — Gemini integration

The AI uses a structured system prompt containing the current session list, student list, and date. It responds with either:

- **Structured JSON** for actions (create session, mark attendance, send QRs)
- **Plain text** for queries (student lookup, absence report)

The app always shows a confirmation dialog before executing any AI action. The AI suggests — the admin decides.

Example commands:
- `"Create a session called Flutter Basics tomorrow at 3pm"`
- `"Mark Ahmed Samir as present in session 2"`
- `"Who was absent in the last session?"`
- `"What is the attendance rate of student 2024015?"`

## Why no Cloud Functions?

Cloud Functions require the Firebase Blaze plan (needs a credit card), even if usage stays within free limits. All backend logic runs client-side in Flutter instead:

| Previously in Cloud Functions | Now in Flutter |
|---|---|
| JWT generation | `dart_jsonwebtoken` on device |
| Email trigger | Direct HTTP call to Resend REST API |
| QR validation | Client validates JWT + Firestore check |
| Sheets write-back | `googleapis` Dart package |

This works perfectly for up to ~100 students and ~10 sessions.

## Creating the first admin

After Firebase setup, create an admin user in the Firebase Console (Authentication tab), then set a custom claim using the Firebase Admin SDK or Cloud Console:

```json
{ "role": "admin" }
```

All other users default to the student role.
